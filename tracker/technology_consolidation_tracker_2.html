Governed AI coding assistance

Claude Enterprise

A bank-approved AI coding assistant for the technology organization, with Claude Code as the primary use case — deployed through Entra ID with full audit logging. Complementary to Microsoft Copilot, not a replacement.

Recommendation: Adopt Claude Enterprise for the technology organization's power users, with Claude Code (and its VS Code extension) as the primary use case. Deploy under SSO and automated provisioning through Entra ID, with audit activity exported to a write-once (WORM) archive. Enterprise is the only Claude tier that clears the governance bar for an FDIC-insured institution — every lower tier fails at least one hard requirement.

One item to flag: disable Claude Cowork at launch (or scope it to a small group). Cowork session activity is currently excluded from Anthropic's audit logs, Compliance API, and data exports on every tier, and its history is stored on the local device — which conflicts with our audit and retention requirements. Our primary use case, Claude Code, is fully covered by audit logging, so this costs us nothing today.

Why not just Microsoft Copilot?

This is a right-tool-for-the-job decision, not a verdict on Microsoft. The two are complementary.

Firm-wide default

Microsoft 365 Copilot

The right regulated assistant for everyone: drafting, summarizing, analyzing documents, and working inside Office and Teams. This decision does not replace it — Copilot should be the assistant in front of all end users.

Coding specialist

Claude Code

An agent that takes a whole task: navigate a repository, read and edit across many files, run commands and tests under explicit permission — from the terminal, VS Code, or desktop. That whole-task capability is what a general chat assistant or autocomplete tool does not provide.

Copilot for everyone; Claude for the people who build and secure our software.

What it costs

Two parts: a flat per-seat access fee, plus token usage billed on top at standard API rates.

Confirmed: Anthropic requires a minimum of 20 Enterprise seats — this is a hard floor, not a starting recommendation, so a "developers only" or "builders only" seat count below 20 isn't purchasable at Enterprise. The seat fee is about $20 / user / month (billed annually) — ~$4,800/year for 20 seats — and covers platform access only. There's no included token allowance and no per-seat usage cap; every token used in chat, Claude Code, or Cowork bills separately at standard API rates, with admin-configurable spend limits per user and per group for predictability.

Why this is a productivity multiplier

This is not simply a more modern ticketing system; it removes the manual work that slows the team down today.

Auto-created

Tickets are created automatically

Alerts from Defender, Rapid7, and Proofpoint open tickets automatically; a GitHub pull request can open and update its own ticket; portal requests are routed automatically by category, priority, and location. The queue populates on its own, freeing technicians to triage and resolve rather than manually log tickets.

Always documented

Every change is documented as it happens

The moment a change is raised, JSM captures who, what, when, the approvals, and the risk level in one ITIL-aligned record. Nothing ships undocumented, and the audit trail is built continuously as work happens rather than reconstructed ahead of an examination.

Assembled for you

Change Management packets compile automatically

The weekly CAB packet — pending changes, approvals, risk ratings, and post-implementation reviews — is generated from the system of record rather than compiled manually in a separate document. The meeting begins from live data.

Deflected

Routine requests resolve through self-service

A modern self-service portal backed by a Confluence knowledge base resolves the repeat requests — password resets, access provisioning, how-tos — before they reach a technician. First-call resolution improves while the manual queue shrinks.

Change Management, automated end to end

Where the manual overhead is heaviest today — and where JSM removes the most work.

The weekly Change Management meeting stops being a manual assembly job. Every change raised in JSM carries its approvals, risk assessment, and post-implementation review with it — so the CAB review and the monthly ISC report are generated from live records rather than rebuilt by hand each week.

Confluence: the knowledge layer

The other half of the Atlassian pair — and where GitHub ties back in.

Confluence is the durable knowledge base behind the self-service portal — answers written once and reused everywhere:

  • Runbooks & architecture decisions — the institutional knowledge that lives in one place, not someone's inbox.
  • Post-implementation reviews link directly back to the change record that produced them.
  • Drives portal deflection and first-call resolution — the engine behind fewer routine tickets.
  • Docs-as-code from GitHub — Markdown in a repo can auto-publish, so docs stay current via the same PR workflow.

Together, JSM and Confluence turn day-to-day tickets and changes into reusable institutional memory.
